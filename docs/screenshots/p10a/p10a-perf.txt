eigenorg — P10a perf re-confirmation (measured, not asserted)
=============================================================

Method: scripted probe in Chromium (Playwright MCP) against the built site
served by scripts/serve.mjs under the /eigenorg/ Pages subpath emulation.
Timings read from the in-page probe hook window.__eigenorg (primaryElapsedMs =
one 500-iteration Monte Carlo run; planElapsedMs = the full 2–3-run plan;
bootReadyMs = navigation start -> Run clickable). Landing preset =
fasterDysfunction (AI on, 3-run plan). Viewport 1280x900.

Re-measured after the round-1 repair (M1/M2/M3/M4+C2/C1 + F1/F2). Numbers are
unchanged materially from the initial build — the fixes are UI-state/a11y only
and add no per-frame work.

Budget A — a 500-iteration sim runs in < 1.5 s:
  run 1: primary 500-iter run = 544 ms
  run 2: primary 500-iter run = 497 ms
  run 3: primary 500-iter run = 530 ms
  full 3-run plan = 1448 ms
  => single 500-iteration sim ~0.52 s  << 1.5 s budget. PASS.

Budget B — a full walkthrough in < 90 s:
  boot (navigation start -> Run clickable) = 401 ms
  scripted walkthrough (land -> open Organization door -> 3 sequential runs
    -> results painted) = 4886 ms (machine-paced proxy; the P5/P6 convention).
  => 4.9 s  << 90 s budget. PASS.

Zero-console-errors + zero-external-requests (same session, Playwright logs):
  console errors (whole session, incl. the malformed-'#s=' decode-error path): 0
  network requests: all to the local /eigenorg/ origin (relative). No CDN, no
    external host; Chart.js served from www/vendor/, wasm from www/pkg via the
    worker. PASS.

Repair-verified behaviours (Playwright, same session):
  - M1: editing a control BEFORE any run keeps #org-stale hidden + charts
    undimmed; after a run, an edit shows the stale badge + dims the charts.
  - M2: cold load leaves document.activeElement = <body> (no focus-on-load) and
    the h1 at the top; Start over focuses the first door.
  - M3: #org-mount / #team-mount report role="region".
  - M4/C2: Start over -> re-enter Org = clean slate (default preset SH 3, setup
    expanded, Run-again hidden, no stale, status "ready…"); Start over DURING a
    run cancels it with no late paint and leaves the "ready…" status.
  - C1: a valid-charset but undecodable '#s=' hash keeps the "could not open the
    shared link…" error status (no "ready…" overwrite).
  - F1: with reduced-motion emulated, .door:hover transform = none (no lift);
    no-preference applies the eased lift.
  - F2: the primary CTA has animationName "none" when unfocused; the breathe is
    scoped to .cta:focus-visible inside the reduced-motion guard.

Notes:
  - The redesign's motion is CSS-only (fade-ups, focus-scoped breathing ring,
    chart-container fade-up); Chart.js per-frame animation stays disabled (P5
    perf discipline), so the restyle adds no per-frame JS animation cost.
  - The chart reveal is a CSS opacity/transform fade-up on the .chart-box
    container (NOT stroke-dashoffset — the charts are Chart.js on canvas).
